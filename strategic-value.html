Asset position

What this asset is

FunnelPlugs is not a website. It is a category artifact. The distinction is structural, not rhetorical.

A website is a property that delivers content to an audience. A category artifact is a property that defines the frame through which an entire professional domain is understood. The Plug Ontology does not describe what funnels do — it classifies what funnels fail to do and why. The Funnel Integrity Standard does not recommend optimization practices — it establishes the structural conditions that determine whether a commercial path is sound. The Protocol does not suggest a workflow — it defines the governed sequence through which structural failure is detected, classified, and addressed. These are not content assets. They are definitional instruments.

The category this asset occupies — structural funnel integrity, commercial path completeness, revenue leak classification — is a space every serious marketing technology company needs vocabulary for. Currently, that vocabulary is fragmented across vendor marketing, informal practitioner language, and generic optimization content. FunnelPlugs exists to be the governed authority layer for that vocabulary. The acquirer of this asset acquires that authority position.

The domain itself carries the structural logic of the asset. "Funnel plugs" is exact-match category terminology — it names the intervention class directly. Combined with a .com extension and the intellectual infrastructure already built on it, the domain is not a starting point for a new asset. It is the address of a category definition that already exists.

The intellectual infrastructure

The domain is the visible surface of the asset. The intellectual infrastructure is what makes the domain valuable. Most domain acquisitions transfer a name. This acquisition transfers a complete, internally consistent, already-published intellectual system.

The Plug Ontology. A proprietary 10-class classification framework for structural failure across commercial paths. Each class is formally defined with structural logic, symptom patterns, required intervention logic, and cross-references to the standard dimensions and protocol stages it maps to. The ontology is the intellectual foundation of the entire asset — the governed vocabulary that makes every other layer of the system coherent. Class names including Trust Infrastructure Failure, Flow Continuity Failure, Recovery Absence Failure, and Measurement Blindness Failure are proprietary terminology established and governed within this system.

The Protocol, Standard, and Manifesto. Three foundational doctrine documents that establish the governing logic of the system. The Manifesto frames the category position. The Standard defines the minimum structural conditions for commercial path integrity across three formal dimensions. The Protocol establishes the three-stage methodology — Detection, Classification, Intervention — through which the standard is applied. Together, these three documents form the doctrine layer of the asset: the intellectual foundation that every derivative work, product feature, or client methodology can reference.

The Revenue Leak Diagnostic Engine. An interactive diagnostic tool at engine.html that operationalizes the ontology for direct user engagement. The engine maps declared funnel conditions against the ontology classes and produces governed diagnostic output: primary leak class identification, integrity band assessment, and intervention priority. Email capture is implemented post-completion — high-intent leads (users who completed 20 diagnostic questions) are captured via a gate before results are delivered. This is a functional tool with an active lead capture mechanism, not static content.

The Structural Integrity Brief. A condensed reference document at structural-integrity-brief.html that covers the three formal integrity dimensions, ten leak classes, diagnostic methodology, and self-assessment criteria. The brief operates as a second lead capture surface — email-gated at the delivery stage, positioned for practitioners who want a working audit reference rather than the full engine diagnostic. The brief and engine serve different user intents: the engine serves active diagnostic need, the brief serves reference and audit workflow need.

The Reference Library. Thirty-one pages of structured, internally linked reference content covering individual leak classes, plug classes, and concept papers. Each page is fully integrated into the site's internal link architecture, governed by the ontology's classification system, and written to the same formal register as the foundational doctrine documents. The reference library provides topical depth, semantic coverage, and SEO surface area across the full lexical space of the category.

Proprietary category terminology. "Funnel plug," "plug class," "leak class," "integrity band," "commercial path completeness" — these terms have been introduced, defined, and governed within this asset. They do not exist as established terminology in competing publications. The acquirer inherits the definitional rights to a terminology set that has no prior established usage outside this system.

SEO and technical infrastructure. Canonical governance across all 31+ pages, structured data markup, sitemap and robots.txt governance, and semantic HTML architecture built for machine-readability. The site has been constructed to function as a reference authority, not merely a content property — a structure that positions it for the kind of citation and link acquisition that compounds category authority over time.

Asset architecture

The asset is built in layers. Each layer has a distinct function. Together they constitute a complete category infrastructure — not a collection of pages, but a governed system with a defined internal logic.

Layer 1 — Doctrine. The Manifesto, Protocol, and Funnel Integrity Standard. These three documents establish the governing logic of the entire system. They define the category position, the methodology, and the structural conditions the system governs. Every other layer references these documents. They are the intellectual constitution of the asset.

Layer 2 — Classification. The Plug Ontology. Ten formally defined failure classes with structural definitions, symptom profiles, required intervention logic, and cross-references to the doctrine layer. The ontology is the governed vocabulary that makes classification precise. Without it, the system would produce descriptions. With it, the system produces structural addresses — specific, actionable, formally named conditions that can be owned, corrected, and monitored.

Layer 3 — Application. The Revenue Leak Diagnostic Engine. A functioning interactive tool that operationalizes the ontology for direct engagement. The engine is the application layer of the asset — the point where the intellectual framework produces utility for a specific user with a specific commercial path. It is also a data surface: every engine interaction produces signal about the distribution of structural failure patterns across the user population, which compounds the asset's research and authority value over time.

This layer provides topical depth and semantic coverage across the lexical space of the category. It is also the primary SEO surface of the asset — the layer through which organic search traffic enters the system and is oriented toward the classification framework and diagnostic engine.

This layer is not visible to users, but it is what makes the asset function as a reference authority rather than a generic content property. It ensures that the intellectual framework and reference library are machine-readable, consistently indexed, and positioned for citation and link acquisition as the category develops.

Each layer is functional independently. Together they operate as a compound system: the doctrine frames the category, the ontology names its failure modes, the engine applies the classification to live paths, the reference library provides depth across the full vocabulary, and the governance layer makes the whole system technically sovereign. An acquirer receives all five layers as a complete, internally consistent asset.
