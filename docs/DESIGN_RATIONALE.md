# DESIGN_RATIONALE.md

## 1) Purpose

This file records design logic for Funnelplugs.com.
Design choices must express doctrine, not decorative preference.

---

## 2) Core Design Principle

Design must reinforce:

- structural clarity
- trust and authority
- controlled cognitive flow
- strategic seriousness

If a design choice weakens any of these, it is rejected.

---

## 3) Rationale Framework

Each meaningful design decision should answer:

1. What strategic function does this serve?
2. What doctrine or architecture rule does it support?
3. What lower-quality alternative was rejected?
4. How does it improve user comprehension and trust?

---

## 4) Current Baseline Rationale

- restrained visual language preserves institutional posture
- clear hierarchy prevents authority dilution
- limited decorative effects reduce noise and preserve legibility
- structured page anatomy supports repeatable expansion
- consistent navigation logic protects reference-system feel

---

## 5) Rejection Rules

Do not approve design changes that:

- prioritize novelty over clarity
- mimic generic martech aesthetics
- increase visual complexity without strategic gain
- weaken reading flow on core pages

---

## 6) Update Rule

Update this file whenever a meaningful visual or UX system decision is made.
If the decision changes governance direction, log it in `DECISION_LOG.md`.
